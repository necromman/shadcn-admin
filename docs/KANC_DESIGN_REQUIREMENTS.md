# KANC 홈페이지 디자인 요구사항 문서

## 📋 요구사항 개요

### SFR-001: 기술원 홈페이지 디자인 제안
- **목적**: 모아팹(MOAFAB) 활용에 친숙하도록 기술원 홈페이지의 UI/UX 디자인 제안
- **주요 요구사항**:
  1. 장비현황, 가동률 등 주요지표 콘텐츠 표시 추가
  2. 기존 한국나노기술원 홈페이지 정보의 시인성 및 가독성 향상
  3. 부서별 담당자 요구사항 분석 후 적용

### SFR-002: 채택 디자인 홈페이지 적용
- **목적**: 다양한 사용자들의 이용환경에 맞추어 서비스 이용이 가능하도록 설계
- **주요 요구사항**:
  1. 이용자 중심의 홈페이지 구축
  2. 사이트 웹 정체성(identity) 표현
  3. 3단계 이하의 페이지 계층 구조
  4. 프레임 미사용
  5. ActiveX 및 비표준 Flash 파일 배제
  6. 시각적 요소와 기능적 요소의 균형

## 📊 주요 지표 콘텐츠 구현 계획

### 1. 장비 현황 대시보드
- **위치**: KANC 서비스 홈페이지 상단
- **표시 항목**:
  - 총 장비 수
  - 사용 가능 장비
  - 점검 중 장비
  - 예약률

### 2. 가동률 시각화
- **구현 방식**: 차트 컴포넌트 (도넛 차트, 라인 차트)
- **표시 데이터**:
  - 일일 가동률
  - 주간 트렌드
  - 장비별 사용 통계

### 3. 실시간 상태 표시
- **구현 요소**:
  - 실시간 뱃지 (Live)
  - 자동 갱신 (5분 간격)
  - 상태별 색상 코드

## 🔧 기술 스택 및 구현 방법

### 사용 기술
- **프론트엔드**: React + TypeScript
- **UI 라이브러리**: shadcn/ui
- **스타일링**: Tailwind CSS
- **차트**: recharts 또는 chart.js
- **상태관리**: Zustand (필요시)

### 구현 우선순위
1. **Phase 1** (즉시): 주요 지표 대시보드 컴포넌트
2. **Phase 2** (1주): 실시간 데이터 연동
3. **Phase 3** (2주): 시각화 및 애니메이션 개선
