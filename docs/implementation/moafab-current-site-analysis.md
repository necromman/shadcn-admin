# MOAFAB 기존 사이트 분석 문서

## 📊 현재 사이트 분석 (https://www.moafab.kr)

### 1. 전체 구조 분석

#### 페이지 구성
- **메인 페이지**: 슬라이더, 공정장비 검색, 공지사항, 알림판
- **서브 페이지**: 팹서비스 소개/신청/현황, 고객센터
- **모바일 대응**: 부분적 반응형 (완전하지 않음)

#### 기술 스택 (추정)
- 서버사이드 렌더링 (JSP 또는 유사 기술)
- jQuery 기반 인터랙션
- 부분적 JavaScript 활용
- 레거시 CSS 구조

### 2. 메인 페이지 구성 요소

#### Header 영역
```
┌─────────────────────────────────────────────────┐
│  로고  │  팹서비스소개 │ 팹서비스신청 │ 팹서비스현황 │ 고객센터  │  로그인  │
└─────────────────────────────────────────────────┘
```

#### 메인 비주얼 (Hero Section)
- **구현**: 이미지 슬라이더 (3개 배너)
- **내용**: 
  - 나노팹 서비스 소개
  - 주요 공지/이벤트
  - 서비스 홍보
- **인터랙션**: 자동 재생, 좌우 네비게이션, 인디케이터

#### 공정장비 검색 섹션
- **레이아웃**: 2단 구성
  - 왼쪽: 기관별 검색
  - 오른쪽: 장비 유형별 검색
- **기능**: 
  - 드롭다운 선택
  - 검색 버튼
  - 결과 페이지 이동

#### 공지사항 & 알림판
- **공지사항**:
  - 최근 5개 게시물 표시
  - 제목, 날짜 표시
  - 더보기 링크
- **알림판**:
  - 배너 슬라이더
  - 이벤트/홍보 콘텐츠

#### 하단 링크 섹션
- 서비스 이용료
- 만족도 조사
- 문의 연락처

#### Footer
- 주소 및 연락처 정보
- 저작권 표시
- 관련 사이트 링크

### 3. 네비게이션 구조 상세

```
MOAFAB
├── 팹서비스 소개
│   ├── 서비스 개요
│   ├── 이용 절차
│   └── 참여 기관
├── 팹서비스 신청
│   ├── 서비스 신청
│   ├── 장비 예약
│   └── 신청 확인
├── 팹서비스 현황
│   ├── 진행 현황
│   ├── 장비 현황
│   └── 통계 정보
└── 고객센터
    ├── 공지사항
    ├── FAQ
    ├── 자료실
    └── 문의하기
```

### 4. UI/UX 특징

#### 장점
- 정보 구조가 명확함
- 주요 기능 접근이 쉬움
- 전문적인 디자인

#### 개선 필요 사항
- **반응형 디자인**: 모바일/태블릿 최적화 부족
- **시각적 계층**: 정보 우선순위 불명확
- **인터랙션**: 사용자 피드백 부족
- **접근성**: 키보드 네비게이션, 스크린리더 지원 미흡
- **성능**: 이미지 최적화 필요
- **현대성**: 구식 디자인 패턴

### 5. 색상 및 디자인 시스템

#### 색상 팔레트
- **Primary**: 파란색 계열 (#0066CC 추정)
- **Secondary**: 회색 계열
- **Background**: 흰색 (#FFFFFF)
- **Text**: 검정/회색

#### 타이포그래피
- 본문: 시스템 폰트
- 제목: Bold 처리
- 크기: 고정 픽셀 단위

#### 레이아웃
- 고정 너비 (1200px 추정)
- 중앙 정렬
- 그리드 시스템 부재

### 6. 기능적 요소

#### 검색 기능
- 공정장비 검색
- 게시판 내 검색
- 통합 검색 부재

#### 사용자 인증
- 로그인/로그아웃
- 회원가입
- 마이페이지 (제한적)

#### 데이터 표시
- 테이블 형식 (게시판)
- 카드 형식 (장비 정보)
- 차트/그래프 부재

### 7. 모바일 경험

#### 현재 상태
- 데스크톱 뷰 축소 형태

- 네비게이션 불편

#### 개선 방향
- 모바일 퍼스트 접근
- 터치 친화적 UI

- 오프라인 지원

#### 문제점
- 이미지 최적화 부족

#### 개선 제안
- 이미지 lazy loading

### 9. 접근성 평가

#### 현재 수준
- 대체 텍스트 부족
- 키보드 네비게이션 제한적
- 색상 대비 일부 미흡
- ARIA 레이블 부재

#### 개선 필요
- WCAG 2.2 AA 준수
- 스크린리더 완벽 지원
- 키보드 전체 접근
- 고대비 모드

### 10. 콘텐츠 구조

#### 정보 아키텍처
- 3단계 깊이
- 논리적 그룹핑
- 일관된 네이밍

#### 콘텐츠 타입
- 정적 페이지 (소개)
- 동적 콘텐츠 (게시판)
- 인터랙티브 요소 (검색, 신청)

### 11. 보안 및 개인정보

#### 현재 적용
- HTTPS 적용

- 개인정보 처리방침
