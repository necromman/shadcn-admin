# LMS 이러닝 서브페이지 구현 계획

## 1. 개요
- **목표**: STEP 사이트를 참고한 이러닝 서브페이지 구현
- **특징**: 왼쪽 카테고리 사이드바 + 오른쪽 본문 (검색 + 테이블 목록)
- **참고 이미지**: docs/image.png

## 2. 페이지 구조

### 2.1 레이아웃 구성
```
┌─────────────────────────────────────────────┐
│                  헤더 (고정)                   │
├──────────────┬──────────────────────────────┤
│              │                              │
│   카테고리    │        과정 검색 영역          │
│   사이드바    │────────────────────────────│
│              │                              │
│   - 전체     │        강의 목록              │
│   - IT/SW    │        (테이블 형식)           │
│   - 디자인    │                              │
│   - 비즈니스  │                              │
│   - 외국어    │                              │
│              │────────────────────────────│
│              │       페이지네이션             │
└──────────────┴──────────────────────────────┘
```

### 2.2 주요 기능
1. **카테고리 필터링**: 좌측 사이드바에서 카테고리 선택
2. **검색 기능**: 과정명, 강사명, 기관명 검색
3. **정렬 기능**: 인기순, 최신순, 평점순
4. **필터링**: 난이도, 수강 상태, 가격대
5. **페이지네이션**: 한 페이지당 20개 표시

## 3. 컴포넌트 설계

### 3.1 파일 구조
```
src/
├── routes/
│   └── elearning.tsx              # 라우트 정의
├── features/
│   └── lms/
│       ├── pages/
│       │   └── ElearningPage.tsx  # 이러닝 페이지
│       └── components/
│           ├── layout/
│           │   └── SubPageLayout.tsx   # 공통 레이아웃
│           ├── elearning/
│           │   ├── CategorySidebar.tsx # 카테고리 사이드바
│           │   ├── CourseSearchBar.tsx # 검색 필터 영역
│           │   └── CourseTable.tsx     # 강의 목록 테이블
│           └── common/
│               └── Pagination.tsx      # 페이지네이션
```

### 3.2 컴포넌트 상세

#### SubPageLayout
- 사이드바 + 메인 콘텐츠 레이아웃
- 반응형 처리 (모바일: 드로어, 데스크톱: 고정)

#### CategorySidebar
- 카테고리 목록 표시
- 선택된 카테고리 하이라이트
- 하위 카테고리 트리 구조

#### CourseSearchBar
- 검색 입력 필드
- 필터 옵션 (드롭다운)
- 정렬 옵션
- 결과 수 표시

#### CourseTable
- 테이블 형식 강의 목록
- 체크박스 선택
- 썸네일 이미지
- 강의 정보 (제목, 강사, 기관, 평점, 수강생 수)
- 수강신청 버튼

#### Pagination
- 페이지 번호
- 이전/다음 버튼
- 페이지당 항목 수 선택

## 6. 구현 단계

### Phase 1: 기본 구조 (30분)
1. 라우트 파일 생성
2. 페이지 컴포넌트 생성
3. 레이아웃 컴포넌트 생성

### Phase 2: 사이드바 구현 (30분)
1. 카테고리 목록 구현
2. 선택 상태 관리
3. 반응형 처리

### Phase 3: 검색 영역 구현 (30분)
1. 검색 바 구현
2. 필터 드롭다운
3. 정렬 옵션

### Phase 4: 강의 목록 구현 (1시간)
1. 테이블 구조
2. 데이터 바인딩
3. 액션 버튼

### Phase 5: 페이지네이션 (30분)
1. 페이지 컴포넌트
2. 상태 관리
3. 이벤트 처리

### Phase 6: 통합 및 최적화 (30분)
1. 컴포넌트 연결
2. 상태 관리
3. 성능 최적화

## 7. 작업 우선순위

1. **필수**
   - 헤더 네비게이션 수정 (이러닝 자식 노드 제거)
   - 라우트 설정
   - 기본 레이아웃
   - 카테고리 사이드바
   - 강의 목록 테이블

2. **권장**
   - 검색 기능
   - 필터링
   - 페이지네이션
   - 반응형 처리

3. **선택**
   - 애니메이션
   - 로딩 상태
   - 에러 처리
