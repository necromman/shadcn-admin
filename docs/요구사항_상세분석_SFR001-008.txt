[SFR-001] 모아팹 홈페이지 개편
----------------------------------------

요구사항 명칭: 모아팹 홈페이지 개편
정의: 모아팹 국문 홈페이지 개편

세부 요구내용:

1. 홈페이지 리뉴얼
   - 웹 접근성 준수 및 웹 표준 준수
   - 사용자 중심의 직관적인 UI 제공
   - 개인정보보호법 준수
   - 콘텐츠 구축 시 필요한 이미지, 사진, 폰트 등은 저작권법을 위배하지 않도록 직접 제작 또는 구매
   - 관리 시스템을 통해서 홈페이지가 유지관리 되어야 함
   - ActiveX 및 비표준 Flash 파일 배제
   - 현재 운영중인 개발환경 및 운영가이드를 준수한 홈페이지 제작

2. 협의체 참여 기관 소개 콘텐츠 추가

3. 협의체 참여 기관 중 자체 홈페이지 없는 기관을 위한 기관별 홈페이지를 클라우드 형태로 제공할 수 있는 기능 추가

4. 기관별 홈페이지 구성(안)
   - 기관별 홈페이지 메인
   - 기관별 소개, 공지, 자료실 게시판 구성
   - 기관별 장비 소개
   - 기관별 팹서비스, 장비별 서비스 신청 기능
   - 신청 내역은 모아팹 운영에서 일괄 처리

산출정보: 홈페이지 설계서

구현방안:
- HTML5/CSS3 기반 웹표준 개발
- W3C WCAG 2.2 가이드라인 준수
- React/Next.js 기반 SPA 구현
- CMS 통합 관리 시스템 구축
- 멀티테넌트 SaaS 아키텍처로 기관별 독립 홈페이지 제공
- AWS 클라우드 인프라 활용

기대효과:
- 웹 접근성 및 사용성 향상
- 기관별 독립적 운영 지원
- 중앙 집중식 관리로 효율성 증대

[SFR-006] 모아팹 시스템 기능 개선 (ETRI 한국전자통신연구원)
----------------------------------------

요구사항 명칭: 모아팸 시스템 기능 개선 (ETRI 한국정보통신연구소)
정의: 1, 2단계 모아팹 시스템 기능 개선

세부 요구내용:

1. 공정별 작업 상세 화면에서 [비고] 항목도 수정할 수 있도록 요청
2. 신청서검토, 담당자승인, 최종승인 메뉴에서 기관을 선택하여 조회할 수 있게 하고, 보여진 Data에 항목에도 기관이 있을 수 있도록 요청
3. Wafer ID도 수량처럼 입력하면 일괄 반영
4. 마이공정에 상세요청내용 작성/저장할 수 있는 기능
5. 서비스신청_공정절차 화면에서 일괄저장 버튼(가장 아래) 표출 화면 고정
6. 모니터링 화면에서 해당 공정이 Hot Run 등 적용 유무 표출
7. 지원/계약 참여기관(기업) 전체 적용 이외에도 특정 계약자로 적용

산출정보: 기능 개선 분석 및 설계서

구현방안:
- 인라인 편집 기능
  * 더블클릭으로 즉시 편집
  * 자동 저장 기능
- 다중 필터링 시스템
  * 기관별 필터
  * 복합 조건 검색
  * 검색 결과 엑셀 다운로드
- Wafer ID 일괄 처리
  * 패턴 인식 자동 생성
  * 범위 지정 입력
  * 중복 검증
- 화면 레이아웃 개선
  * 고정 헤더/푸터
  * 스크롤 유지
  * 반응형 버튼 배치
- 상태 표시 강화
  * Hot Run 뱃지 표시
  * 색상 코드 구분
  * 실시간 상태 업데이트

기대효과:
- 작업 효율성 대폭 향상
- 데이터 입력 시간 단축
- 모니터링 가시성 개선

[SFR-008] 선수금 관리 기능 개선
----------------------------------------

요구사항 명칭: 선수금 관리 기능 개선
정의: 모아팹활용지원사업, 바우처 사업 등 선수금 관리 개선

세부 요구내용:

1. 모아팹활용지원사업에 선정된 사용자(연구책임자) 기준으로 모아팹에서 서비스를 신청하고, 선정된 비용만큼 기관의 정산 내용과 연계하여 처리할 수 있도록 기능 추가

2. 모아팹활용지원사업의 연구책임자가 참여 연구자를 관리하고, 참여자도 동일한 연구책임자의 선정 내용으로 서비스를 신청할 수 있도록 기능 구현

3. 연구책임자는 참여 연구자가 사용한 내역을 확인하고 관리할 수 있어야 함

4. 기관에서 발행한 바우처로 사용자가 서비스를 신청할 경우 참여기관(기업) 외에도 특정 사용자가 사용할 수 있도록 개선

산출정보: 기능 개선 분석 및 설계서

구현방안:
- 계층적 권한 관리
  * 연구책임자 - 참여연구자 구조
  * 권한 위임 기능
  * 승인 워크플로우
- 예산 관리 시스템
  * 프로젝트별 예산 할당
  * 실시간 잔액 확인
  * 사용 한도 설정
- 정산 연계 자동화
  * 사용 내역 자동 집계
  * 정산 문서 자동 생성
  * 승인 프로세스 연동
- 바우처 관리 고도화
  * 바우처 발급/회수
  * 사용자별 할당
  * 유효기간 관리
  * 사용 이력 추적

기대효과:
- 연구비 관리 투명성 확보
- 정산 업무 자동화
- 예산 집행 효율성 향상
- 부정 사용 방지
